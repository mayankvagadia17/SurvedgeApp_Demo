ARCHITECTURE SUMMARY
================================================================================

Design Pattern: State-Driven UI (Jetpack Compose)

Data Flow:
  User taps Code row
    → state.onSelectCode() callback triggered
    → Fragment logic handles callback
    → Fragment shows SelectCodeSheet
    → User selects code
    → Fragment updates EditLineSheet state with new code
    → EditLineSheet recomposes with new code value
    → Code display updates automatically
    → User saves
    → ViewModel.saveLine() persists to database

Benefits:
  ✅ No counters or race conditions
  ✅ Single source of truth (StateFlow)
  ✅ Declarative UI (describes state, not commands)
  ✅ Recomposable (updates automatically)
  ✅ Type-safe (sealed class prevents invalid states)
  ✅ Testable (pure functions, no side effects in composables)
