# Mapping Overlays

[Index](../../index.md) > [Mapping](../readme.md) > Overlays

Custom OSMDroid overlay classes used in the Survedge mapping module to handle complex visuals and interactions.

---

## Layering and Interaction Priority

The map overlays are managed to maintain a clear visual hierarchy and ensure that UI-blocking interactions (like panels and buttons) are respected. The general z-order (from bottom to top) is:

1.  **`RotationGestureOverlay`**: Handles two-finger rotation; typically at the base of the stack.
2.  **`DashedPolylineOverlay` / `ClickablePolylineOverlay`**: Lines representing project data or connection indicators in stakeout.
3.  **`Marker` (Points)**: Survey points with icons and labels, placed above lines.
4.  **`BullseyeOverlay`**: Precision target interface, shown only during specific stakeout phases (hides other layers when active).
5.  **`DoubleTapInterceptorOverlay`**: Prevents double-tap zoom through UI panels.
6.  **`PointClickHandlerOverlay`**: Handles general map taps; kept at the absolute end (top) of the overlay list to catch taps that miss markers/lines.

## Overlay Documentation

### BullseyeOverlay
*   **Purpose**: Provides a high-precision "Precision Mode" visual for stakeout, centering the target and showing a relative "Rover" position.
*   **What it draws**: Static target ring (340px ring), crosshairs, a moving "Rover" circle with an "R" label, and a directional "Arch" pointer when the rover is off-screen.
*   **Touch Events**: None (Purely for visualization).
*   **Lifecycle**: Added/Removed via `MappingFragmentHelper.showBullseyeView()` and `hideBullseyeView()`.

### ClickablePolylineOverlay
*   **Purpose**: A subclass of `Polyline` that allows line segments to be selectable and highlighted.
*   **What it draws**: A standard polyline, potentially dashed or closed, with metadata-driven colors.
*   **Touch Events**: Overrides `onSingleTapConfirmed` to check if a tap occurs within a proximity threshold (measured in pixels) of the line segments.
*   **Lifecycle**: Created for each project line. Added below markers via `addPolylineBelowMarkers()` logic to respect visual priority.

### DashedPolylineOverlay
*   **Purpose**: A lightweight utility for drawing dashed lines without the overhead of the full Polyline class features.
*   **What it draws**: A `Path` with a `DashPathEffect`.
*   **Touch Events**: None.
*   **Lifecycle**: Used for temporary visual aids, such as the connection line between the user's location and the stakeout target.

### DoubleTapInterceptorOverlay
*   **Purpose**: Prevents the map from zooming when the user double-taps on UI elements that overlay the map (e.g., side panels, buttons).
*   **What it draws**: Nothing.
*   **Touch Events**: Intercepts `onTouchEvent` and checks if the touch point is within any `protectedViews`. If so, uses a `GestureDetector` to consume double-tap events.
*   **Lifecycle**: Initialized with a list of "protected" views during fragment setup.

### LabeledPointOverlay
*   **Purpose**: Originally intended to handle points with labels. 
*   **Note**: The class itself is commented out in `LabeledPointOverlay.kt`. The project currently uses `LabeledPoint` as a data model, and `MappingFragmentLogic` manages the lifecycle of standard `Marker` objects with icons generated by `createLabeledPointBitmap`.
*   **What it draws**: A point marker icon (dot) and text labels (ID and Code).

### PointClickHandlerOverlay
*   **Purpose**: Captures taps on the map to handle point selection or general map clicks, while filtering out interactions with protected UI elements.
*   **What it draws**: Nothing.
*   **Touch Events**: `onSingleTapConfirmed` checks against `protectedViews`. If the tap is on the map, it invokes the `onPointClick` callback with the clicked `GeoPoint`.
*   **Lifecycle**: Added to the map during fragment initialization; order is ensured by `ensurePointClickHandlerAtEnd()`.

### RotationGestureOverlay
*   **Purpose**: Implements smooth two-finger map rotation.
*   **What it draws**: Nothing.
*   **Touch Events**: Handles multi-touch gestures. It calculates the rotation delta between fingers, applies exponential smoothing to reduce jitter, and updates `mapView.mapOrientation`.
*   **Lifecycle**: Added at index 0 during `initializeMap`. Can be enabled/disabled (e.g., disabled during Bullseye mode).
